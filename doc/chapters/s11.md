# s11：Error Recovery

本章加入错误分类、有限重试和恢复状态展示。目标不是自动掩盖失败，而是把不同错误
归类，用保守策略恢复，并向用户和模型提供清晰结果。

## 数据流

```text
Agent model call
    |
    v
RetryPolicy retries ModelError only
    |
    v
Tool / Permission / Hook results remain tool results
    |
    v
App records recent errors
    |
    v
/status shows latest error summary
```

写工具、权限拒绝和 Hook 阻断不会被自动重试。工具失败继续作为 ToolResult 回填给
模型，让模型决定下一步。

## 错误分类

`internal/recovery` 定义：

- `ModelError`
- `ToolError`
- `PermissionError`
- `HookError`
- `CompactError`
- `StoreError`
- `ConfigError`

`Wrap` 用于显式包装错误，`Classify` 能从 typed error 或错误文本中推断类别。
`Summary` 用于 `/status` 展示。

## RetryPolicy

`RetryPolicy` 包含：

- `MaxAttempts`
- `Backoff`
- `ShouldRetry`
- `BackoffFor`

s11 默认只对 `ModelError` 做有限重试。GoClaw 不自动重试权限拒绝、Hook 阻断或结果
未知的写工具，避免把安全问题和副作用操作变成重复执行。

## 核心实现

### 1. Agent 恢复

`internal/agent/loop.go` 在模型调用外增加 `runModelWithRetry`：

- 模型请求失败且未输出内容时，可以按 `RetryPolicy` 重试。
- 超过上限后返回清晰的 `ModelError`。
- 如果模型已经流式输出部分内容，则不重试，避免重复文本或上下文混乱。

工具错误仍通过 `toolResultText` 回填给模型。权限拒绝由 permission result 进入工具
结果；Hook 阻断由 Hook blocked 工具结果进入上下文，不继续执行真实工具。

### 2. App 状态

`internal/app/errors.go` 新增内存内 `ErrorRecorder`。App 会记录：

- Agent 运行错误。
- context compact/store 失败。
- 审批持久化失败。
- 回复流关闭或追加失败。

`/status` 现在包含最近错误摘要。没有错误时显示 `none`。

## 关键文件

- `internal/recovery/errors.go`：错误类别、包装、分类和重试策略。
- `internal/agent/loop.go`：模型失败有限重试。
- `internal/app/errors.go`：最近错误记录器。
- `internal/app/app.go`：记录错误并在 `/status` 展示。
- `internal/agent/loop_test.go`：模型重试和超过上限测试。
- `internal/app/app_test.go`：最近错误状态测试。

## 测试

运行：

```bash
go test ./...
```

s11 的测试覆盖：

- typed error 分类。
- RetryPolicy 只重试 ModelError。
- 模型失败后重试成功。
- 模型超过重试上限后返回 ModelError。
- 工具失败保持 ToolResult 回填。
- 权限拒绝不重试危险操作。
- Hook 阻断不继续执行工具。
- Compact 失败不让 App 崩溃。
- `/status` 显示最近错误摘要。

## 面试解释

错误恢复要先区分“能不能安全重试”。模型网络错误通常是幂等的，可以有限重试；
写文件、写 memory、权限拒绝和 Hook 阻断则不能自动重试，否则可能重复副作用或绕过
安全意图。

GoClaw 的做法是把错误分类作为统一语言：Agent 用它决定模型重试，App 用它向用户
展示最近错误。这样既能恢复暂时性模型失败，也能让权限、Hook、compact 和 store
问题保持可见。
