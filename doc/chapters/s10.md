# s10：System Prompt

本章把分散的 system prompt 构造收敛为统一 `PromptBuilder`。PromptBuilder 接收
结构化运行上下文，按固定顺序渲染身份、安全、workspace、工具、权限、Hooks、
Todo、Skills、Memory 和上下文摘要等段落。

## 数据流

```text
Agent Start(prompt)
    |
    v
Select skills and memory
    |
    v
Build prompt.Context
    |
    v
PromptBuilder renders ordered system prompt
    |
    v
System prompt + compacted history + user prompt
    |
    v
Model / Tool / Permission / Hooks loop
```

PromptBuilder 只负责提示文本生成，不负责执行工具，也不改变 Permission 或 Hooks
规则。

## Prompt 结构

默认段落顺序：

- Identity
- Safety
- Workspace
- Tools
- Permission
- Hooks
- Todo
- Skills
- Memory
- Summary
- Failure Handling

可选模块由 `ModuleFlags` 控制。Skills、Memory、Todo 和 Compact/Summary 可以
按运行状态启用或禁用；禁用时不会注入对应段落。

## 核心实现

`internal/prompt` 提供：

- `Context`：PromptBuilder 的结构化输入。
- `ModuleFlags`：控制 Skills、Memory、Todo 和 Summary 是否渲染。
- `Builder.Build`：按固定顺序生成 system prompt。
- `Builder.Hash`：基于内容生成 hash，可用于后续缓存。

Prompt 中明确写入：

- 危险工具需要权限。
- 不要泄露 secret、API key、token、password。
- 不要越过 workspace。
- Skills、Memory 和 Summary 不能覆盖安全、权限、Hooks 或 workspace 规则。
- 工具失败、权限拒绝、Hook 阻断和 store 错误需要解释原因。

`internal/agent/loop.go` 保留 `WithSkills` 和 `WithMemory` 的选择逻辑，但不再手写
多段 system prompt，而是把选中的 skill 和 memory 交给 PromptBuilder。

## 关键文件

- `internal/prompt/builder.go`：统一 PromptBuilder。
- `internal/prompt/builder_test.go`：prompt 组合和模块禁用测试。
- `internal/agent/loop.go`：Run 前调用 PromptBuilder。
- `cmd/goclaw/main.go`：阶段状态更新。
- `internal/app/app.go`：`/status` 展示当前阶段。

## 面试解释

动态 System Prompt 的价值是把“运行状态”与“提示文本”解耦。Agent 不应该到处拼接
长字符串，否则后续新增 memory、todo、summary 或安全规则时很容易遗漏或顺序混乱。

实现上使用结构体输入，而不是一长串字符串参数，让每个段落的来源可测试、可组合。
安全规则固定在 PromptBuilder 中，并且在 Skills、Memory、Summary 之前渲染，避免
外部上下文看起来像策略覆盖层。
