# s02：Tool Use

本章把 s01 的单一 bash 扩展为统一工具系统，并加入受工作区限制的结构化文件工具。

## 数据流

```text
AgenticModel
    |
    v
FunctionToolCall[]
    |
    v
ToolRegistry
    |----> 连续只读批次：并发执行
    |----> 写工具 / bash：逐个执行
    |
    v
按原始调用顺序排列的 FunctionToolResult[]
```

Agent Loop 仍然负责“模型调用 -> 工具结果回填 -> 再调用”。工具查找、参数验证、
并发分批和执行错误由 `internal/tool` 负责。

## 核心实现

### 1. Tool 与 Registry

每个工具实现：

- `Info()`：返回 Eino `schema.ToolInfo`。
- `ConcurrencySafe()`：说明调用是否可与相邻安全调用并发。
- `Run()`：执行经过工具级校验的调用。

`Registry` 在启动时拒绝空工具和重复名称，并保持注册顺序向模型传递 Schema。模型
返回多个工具调用时，Registry 按原始顺序切分连续批次：

```text
[read A, glob *.go, write B, read C]
    -> 并发 [read A, glob *.go]
    -> 串行 [write B]
    -> 并发 [read C]
```

批次内的只读结果即使完成顺序不同，回填顺序仍与模型调用顺序一致。s02 将
`read_file` 和 `glob` 标记为并发安全；`write_file`、`edit_file` 和 `bash` 顺序
执行。bash 暂不分析命令是否只读。

### 2. 统一参数校验

工具参数同时有两层约束：

- Eino Tool Schema 告诉模型字段类型和必填字段。
- Go 端严格 JSON 解码，拒绝未知字段、多余 JSON 值、缺少字段和非法值。

校验错误不会让 Agent Loop 崩溃，而是以 `Error: ...` 工具结果回填给模型，使模型
可以修正参数后继续执行。

### 3. 路径沙箱

文件工具只接受工作区相对路径：

- 拒绝绝对路径、空路径和包含 `..` 的路径。
- 已存在路径先解析符号链接，再确认最终路径仍位于工作区。
- 新文件解析最近的已存在父目录，防止通过父目录符号链接写到工作区外。
- glob 使用 `WalkDir`，不会跟随目录符号链接；匹配到逃逸符号链接时返回错误。

这些检查覆盖确定性的路径逃逸，但不是操作系统级沙箱。并发修改符号链接产生的
TOCTOU 风险和 bash 的完整隔离不属于本章。

### 4. 文件工具

`read_file`

- 读取工作区文本文件。
- 可用 `limit` 限制返回行数，并提示剩余行数。

`write_file`

- 写入完整文件内容。
- 自动创建父目录。
- 使用同目录临时文件、`fsync` 和重命名替换，避免留下半写文件。

`edit_file`

- 要求 `old_text` 非空且在文件中恰好出现一次。
- 零次匹配或多次匹配都拒绝修改。
- 使用与 `write_file` 相同的原子写入方式。

`glob`

- 支持 `*`、`?`、字符类和 `**`。
- 结果使用工作区相对路径并按字典序返回。

## 测试

离线测试覆盖：

- 五个工具的 Schema 和端到端调用。
- 父目录、绝对路径和符号链接逃逸。
- 嵌套目录写入、读取行数限制和原子替换。
- 精确编辑的零匹配、唯一匹配和多匹配。
- glob 的根目录与递归匹配。
- 连续只读工具真实并发、写工具串行和结果保序。
- s01 bash、取消和 Agent Loop 行为保持兼容。
