# s00：工程启动

本章建立 GoClaw 的运行外壳，但暂不实现 Agent Loop。

## 数据流

```text
CLI / 飞书
    |
    v
Channel.Message
    |
    v
app.Handle
    |----> Event ID 去重
    |----> 命令路由
    |----> 会话状态
    v
Channel.Stream
```

业务层只认识 `channel.Message` 和 `channel.Stream`，不知道消息来自终端还是飞书。
后续 s01 加入 Agent Loop 时，只需要替换普通消息的处理逻辑，不需要修改 IM 接入。

## 学习目标

- 理解为什么 IM、业务逻辑、状态存储和模型客户端需要通过接口隔离。
- 使用 `context.Context` 和系统信号控制进程生命周期。
- 使用原子写入保存可直接阅读的 JSON 状态。
- 使用 Fake Channel 编写不依赖飞书网络的测试。

## 核心实现

### 1. 配置

`internal/config` 只负责读取和验证环境变量：

- 工作区必须真实存在，并先解析符号链接。
- CLI 不要求模型和飞书凭据。
- 选择飞书时，App ID、App Secret 和用户白名单必须存在。
- 开启群聊时必须同时配置群白名单。

把校验集中在启动阶段，可以避免程序运行一段时间后才发现配置不完整。

### 2. 文件存储

`internal/store` 使用两类文件：

- `events/<sha256>.json`：通过 `O_CREATE|O_EXCL` 保证一个 Event ID 只被首次记录。
- `sessions/<sha256>.json`：保存会话代次和状态。

文件名使用原始 ID 的 SHA-256，避免 IM 中的 `/` 等字符逃逸目录。会话状态先写入
同目录临时文件，执行 `fsync` 后再原子重命名，防止写到一半留下损坏 JSON。

### 3. Channel

`internal/channel` 定义三个重要接口：

- `Channel`：启动和关闭传输层。
- `Responder`：为某条入站消息创建回复。
- `Stream`：追加文本并结束流式回复。

实现包括：

- CLI：一行输入对应一条消息。
- Fake：在内存中记录回复，用于离线测试。
- 飞书：包装官方 SDK 的 `Channel` WebSocket 模块。

飞书 SDK 将空群白名单解释为“允许所有群”。GoClaw 在关闭群聊时放入一个不可能
匹配的内部哨兵值，从而保持“群聊默认关闭”的安全语义。对应行为有离线测试保护。

### 4. 命令路由

`internal/app` 实现：

- `/help`：显示命令。
- `/status`：读取会话状态。
- `/new`：取消旧任务并增加会话代次。
- `/cancel`：取消当前运行中的任务。

`RunRegistry` 为每次运行分配唯一 ID。旧任务的延迟清理只能删除自己的记录，不能
误删同一聊天中新启动的任务。

### 5. 生命周期

`internal/server` 在 goroutine 中启动 Channel，主 goroutine 等待：

- Channel 返回错误；或
- 系统信号取消根 `context.Context`。

退出时最多等待五秒关闭 Channel。所有未来的 Agent 和工具执行都必须继承该根
Context，才能响应 `/cancel` 和进程退出。

### 6. Eino 模型工厂

`internal/llm` 使用：

- Eino `v0.9.1`
- `agenticopenai v0.2.1`

工厂返回 `model.AgenticModel`。它允许在每次请求时通过 `model.WithTools` 传入工具，
不会像旧版 `BindTools` 一样修改共享模型实例。s00 只创建客户端，不发送模型请求。

## 阅读顺序

1. `internal/channel/channel.go`
2. `internal/app/app.go`
3. `internal/store/file.go`
4. `internal/server/server.go`
5. `cmd/goclaw/main.go`
6. 对应的 `_test.go`

建议先画出接口之间的依赖箭头，再阅读具体飞书 SDK 代码。
