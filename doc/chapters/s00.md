# s00：工程启动

本章建立 GoClaw 的运行外壳，但暂不实现 Agent Loop。

## 学习目标

- 理解为什么 IM、业务逻辑、状态存储和模型客户端需要通过接口隔离。
- 使用 `context.Context` 和系统信号控制进程生命周期。
- 使用原子写入保存可直接阅读的 JSON 状态。
- 使用 Fake Channel 编写不依赖飞书网络的测试。

## 本章验收

- CLI 可以接收消息并响应基础命令。
- 飞书 Channel 可以通过官方 SDK 长连接接入。
- Event ID 可以跨进程持久化去重。
- `/help`、`/status`、`/new`、`/cancel` 有确定行为。
- OpenAI 兼容模型工厂可以从配置创建 Eino `AgenticModel`。
- `go test ./...`、`go test -race ./...` 和 `go vet ./...` 全部通过。
