# s09：Memory

本章加入长期 Memory。Memory 用于保存跨会话仍有价值的用户偏好、项目事实、
反馈和参考信息，并在下一轮 Agent Run 前按关键词选择相关条目注入上下文。

## 数据流

```text
Startup
    |
    v
Open .goclaw/memory/
    |
    v
Register memory_read and memory_write tools
    |
    v
Agent Start(prompt)
    |
    v
Select relevant memory by keywords
    |
    v
Inject long-term memory system message
    |
    v
Model may call memory_read or memory_write
    |
    v
memory_write must pass Permission ask and user approval
```

Memory 注入只是上下文材料，不是安全策略。它不能覆盖 workspace、Permission、
Hooks 或工具规则。

## Memory 文件

默认目录：

```text
.goclaw/memory/
```

s09 支持四类 Markdown 记忆：

- `user.md`：用户偏好和长期约束。
- `feedback.md`：用户对结果的反馈。
- `project.md`：项目事实和架构约定。
- `reference.md`：可复用参考信息。

每条 memory 保存为 Markdown section，包含 `id`、`created_at`、`updated_at` 和
quoted `content`。达到本地阈值后，旧条目会进入 `archive/`，不是直接删除。

## 工具

### memory_read

`memory_read` 是只读工具，支持 `query`、`category` 和 `limit`：

```json
{
  "query": "fake model tests",
  "category": "project",
  "limit": 5
}
```

返回选中的长期记忆，以及明确的安全边界说明。

### memory_write

`memory_write` 用于新增或更新长期记忆：

```json
{
  "category": "user",
  "id": "answer-style",
  "content": "prefers direct answers"
}
```

该工具会持久化数据，所以必须经过 s03 Permission 的 `ask`，再由用户审批。敏感内容
会在工具校验阶段被拒绝，包括 API key、token、password、secret、精确地址和身份
证号等。

## 核心实现

`internal/memory` 提供：

- `Entry` 和 `Category`：四类 memory 的基础模型。
- `Store`：以 Markdown 文件持久化，原子写入，文件权限 `0600`。
- `Select`：关键词选择器，不依赖 embedding 或真实模型。
- `FormatPrompt`：把选中记忆渲染为 system prompt 片段。
- `DetectSensitive`：保守的敏感信息检测。

`internal/tool/memory.go` 提供 `memory_read` 和 `memory_write`。`memory_write` 仍是
普通工具，所以会经过 ToolRegistry、Permission、Hooks 和 Agent 工具结果回填。

`internal/agent/loop.go` 通过 `WithMemory` 在 Run 前选择相关 memory，并把结果注入
system message。没有匹配 memory 时，Agent 正常运行。

## 关键文件

- `internal/memory/model.go`：memory 类型和分类校验。
- `internal/memory/store.go`：Markdown store、upsert、归档和选择。
- `internal/memory/selector.go`：关键词匹配和 prompt 渲染。
- `internal/memory/sensitive.go`：敏感信息检测。
- `internal/tool/memory.go`：`memory_read` 和 `memory_write` 工具。
- `internal/permission/engine.go`：`memory_write` 需要人工审批。
- `internal/agent/loop.go`：Run 前注入选中 memory。
- `cmd/goclaw/main.go`：初始化 `.goclaw/memory` 并注册工具。

## 面试解释

Memory 和 Context Compact 的区别在于生命周期。Context 是当前会话历史，会被压缩；
Memory 是长期事实，跨会话保留，并且应该更谨慎地写入。

实现上最重要的是安全默认值：读 memory 可以自动执行，写 memory 必须审批，疑似
敏感内容直接拒绝。这样能避免模型把 token、password、地址或身份信息长期保存。
Memory 注入也明确不能覆盖安全和权限规则，防止“提示即策略”的边界混淆。
