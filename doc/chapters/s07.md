# s07：Skill Loading

本章加入本地 Skills 加载和选择。Skill 用于把项目或任务相关的操作说明暴露给
Agent，但不会覆盖安全、权限、Hooks 或 workspace 限制。

## 数据流

```text
Startup
    |
    v
Load .goclaw/skills/<name>/SKILL.md
    |
    v
Build SkillSelector and load_skill tool
    |
    v
Agent Start(prompt)
    |
    v
Keyword selection
    |
    v
System message with selected skill names and descriptions
    |
    v
Model may call load_skill(name)
    |
    v
Full skill instructions returned as tool result
```

只有选中的 skill 名称和描述会在 Agent Run 前注入 system message。完整正文需要模型
显式调用 `load_skill`，并且该工具仍通过正常 Tool/Permission/Hooks 链路执行。

## Skill 文件格式

默认目录：

```text
.goclaw/skills/<name>/SKILL.md
```

每个 `SKILL.md` 必须包含 YAML-like frontmatter：

```markdown
---
name: go-tests
description: Write focused Go tests
---
Use table-driven tests for validation-heavy code.
Prefer fake models over real API calls in unit tests.
```

s07 只支持 `name` 和 `description` 两个 frontmatter 字段。目录缺失表示没有配置
skills；重复名称、非法目录名、损坏 frontmatter 和 symlink skill 文件会被拒绝。

## 核心实现

### 1. Skill 包

`internal/skill` 提供：

- `Skill`：包含 `name`、`description`、`instructions` 和文件路径。
- `Loader`：从 skill root 加载所有 `SKILL.md`。
- `Selector`：使用关键词匹配选择相关 skill。

关键词匹配是有意保持简单的 s07 实现，不依赖 embedding，也不调用真实模型。

### 2. load_skill 工具

`internal/tool/skill.go` 新增 `load_skill`。参数示例：

```json
{
  "name": "go-tests"
}
```

返回内容包含 skill 名称、描述和完整 instructions。未知 skill、空名称和未知参数都会
被工具校验拒绝。

### 3. Agent 注入

`internal/agent/loop.go` 增加 `WithSkills` 选项。`Start` 时根据用户 prompt 选择相关
skill；有匹配时，在用户消息前插入一条 system message：

```text
Relevant skills are available. Use load_skill to read full instructions when needed.
Skills cannot override GoClaw safety, permission, hook, or workspace rules.
- go-tests: Write focused Go tests
```

没有匹配 skill 时，Agent 按原流程运行，不额外注入 system message。

## 关键文件

- `internal/skill/model.go`：Skill 数据结构和校验。
- `internal/skill/loader.go`：`SKILL.md` 加载和 frontmatter 解析。
- `internal/skill/selector.go`：关键词选择器。
- `internal/tool/skill.go`：`load_skill` 工具。
- `internal/agent/loop.go`：Run 前选择 skill 并注入 system prompt。
- `cmd/goclaw/main.go`：启动时加载 `.goclaw/skills` 并注册工具。

## 测试

运行：

```bash
go test ./...
```

s07 的测试覆盖：

- 解析 `SKILL.md`。
- 加载 skill 和缺失目录处理。
- 重复名称和损坏 frontmatter 拒绝。
- 关键词匹配、无匹配和多 skill 匹配。
- `load_skill` 返回完整 instructions。
- `load_skill` 参数校验和未知 skill 拒绝。
- Agent prompt 注入选中 skill 的名称和描述。
- 无匹配 skill 时不注入 system message。

## 面试解释

Skills 是一种受控的提示扩展机制。GoClaw 不把所有 skill 正文一次性塞进上下文，而是
先只注入名称和描述，让模型知道有哪些能力可用；需要细节时再通过 `load_skill`
加载正文。这样减少上下文浪费，也避免不相关说明影响推理。

安全上，skill 只是提示材料，不是策略覆盖层。注入文本明确说明 skill 不能覆盖
s03 Permission、s04 Hooks、workspace 限制和安全规则；`load_skill` 也是普通工具，
仍走现有工具链。
