# s08：Context Compact

本章加入会话历史持久化和上下文压缩。GoClaw 会按 chat/session 保存 Agent
消息历史，并在超过阈值时生成 summary，只把 summary 和最近消息带入下一轮。

## 数据流

```text
App receives prompt
    |
    v
Load compacted history from .goclaw/context/
    |
    v
Agent StartWithHistory(history, prompt)
    |
    v
Model / Tool / Permission / Hooks loop
    |
    v
RunResult.Checkpoint exposes final messages
    |
    v
Convert messages to context history
    |
    v
Context Manager compacts when policy threshold is exceeded
    |
    v
Save history back to .goclaw/context/
```

上下文压缩在 Agent 运行前后发生，不改变工具执行链路。工具调用仍然通过
ToolRegistry、Permission 和 Hooks；压缩只影响后续模型能看到的历史消息。

## 核心实现

### 1. ConversationHistory

`internal/contextmgr` 定义持久化消息模型：

- `ConversationHistory`：按 `session_id` 保存消息列表。
- `Message`：包含 `role`、`content` 和 `created_at`。
- `CompactPolicy`：包含最大消息数、最大字符数和保留最近消息数。

默认策略为 40 条消息、32 KiB 字符预算、保留最近 8 条消息。

### 2. Summarizer 接口

`Summarizer` 接收旧消息、前次 summary 和 Todo 摘要，返回新的压缩 summary。
s08 默认实现 `RuleSummarizer`，它是确定性的本地 summarizer，适合单元测试和无
模型场景。后续可以替换为 ModelSummarizer，但测试不依赖真实 API Key。

summary 会保留关键运行信号，包括：

- 旧的 summary。
- 当前 Todo 状态摘要。
- 工具结果。
- 用户和助手的关键文本。

### 3. App 和 Agent 接入

`agent.Runner` 新增 `StartWithHistory`，把 `RoleSummary` 转为 system message，把
近期 user/assistant/tool 消息恢复为模型上下文。完成态 `RunResult` 会携带
checkpoint，便于 App 保存本轮消息。

`app.App` 通过 `SetContextManager` 启用上下文存储：

- 普通 prompt 运行前按 chatID 读取历史。
- 运行结束后从 checkpoint 转换消息并保存。
- 达到策略阈值时调用 manager compact。
- summarizer 失败时保存未压缩历史，不让 Agent 整体失败。

## 关键文件

- `internal/contextmgr/model.go`：历史消息、角色和压缩策略。
- `internal/contextmgr/manager.go`：追加消息、触发压缩和保留最近消息。
- `internal/contextmgr/summarizer.go`：`Summarizer` 接口和本地规则摘要。
- `internal/contextmgr/store.go`：按 session/chatID 持久化 JSON 历史。
- `internal/agent/loop.go`：`StartWithHistory` 和 Agent 消息转换。
- `internal/app/app.go`：运行前读取历史、运行后保存和压缩。
- `cmd/goclaw/main.go`：初始化 `.goclaw/context` store 和默认 manager。

## 测试

运行：

```bash
go test ./...
```

s08 的测试覆盖：

- 未达到阈值时不压缩。
- 达到消息数或字符阈值时触发压缩。
- 压缩后保留 summary 和最近消息。
- summary 保留工具结果、关键文件、错误和 Todo 摘要。
- summarizer 失败时返回原始历史。
- 历史按 session/chatID 持久化并隔离。
- Agent Run 前注入 summary 和近期历史。
- Agent Run 后 App 保存 checkpoint 消息。
- App 压缩失败时不影响本次回复。

## 面试解释

上下文压缩的核心是“完整历史可持久化、模型上下文可裁剪”。GoClaw 不把全部历史
无限塞给模型，而是把旧消息压缩成 summary，并保留最近几条原始消息，兼顾连续性
和上下文预算。

实现时要强调边界：压缩不是权限层，也不是工具层。它只在 Agent Run 前后处理消息
上下文，不会绕过 s03 Permission 或 s04 Hooks。失败策略也要保守，summary 失败时
保存未压缩历史并记录日志，避免因为压缩失败让用户任务直接崩溃。
