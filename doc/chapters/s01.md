# s01：Agent Loop

本章在 s00 的 Channel、App、Store 和 LLM 边界上加入最小可用 Agent Loop，并提供
一个受限 `bash` 工具。

## 数据流

```text
Channel.Message
    |
    v
App.Handle
    |----> RunRegistry.Begin(chatID)
    v
agent.Runner
    |----> AgenticModel.Stream(messages, model.WithTools(...))
    |           |
    |           +----> 普通文本 ----> Channel.Stream.Append
    |           |
    |           +----> bash 调用 ----> BashTool.Run
    |                                  |
    |                                  +----> FunctionToolResult
    |                                              |
    +----------------------------------------------+
```

模型返回最终文本时运行结束；模型请求 `bash` 时，Agent 执行命令、回填工具结果并
继续调用模型。每次模型调用计为一个 step，达到上限后停止。

## 核心实现

### 1. Agent Loop

`internal/agent/loop.go` 直接使用 Eino `model.AgenticModel`：

- 用户消息转换为 `schema.UserAgenticMessage`。
- 每次请求通过 `model.WithTools` 传入工具 Schema。
- `AssistantGenText` 内容块立即转发给 Channel。
- `FunctionToolCall` 按顺序执行，并转换为 `FunctionToolResult`。
- 未注册的工具不会执行，而是把明确错误作为工具结果回填。
- 空响应或持续工具调用最多运行 `GOCLAW_MAX_STEPS` 步。

当前不会持久化消息历史。一次用户请求内的模型和工具消息只存在于该次运行中。

### 2. bash 工具

`internal/agent/bash.go` 使用 `bash -lc` 在配置的工作区执行命令：

- 单次执行受 `GOCLAW_BASH_TIMEOUT` 限制。
- stdout 和 stderr 合并，并受 `GOCLAW_BASH_OUTPUT_LIMIT` 限制。
- Agent 取消时终止 bash 进程组。
- 非零退出状态和超时作为工具结果返回给模型。
- 明确拒绝根目录递归删除、关机、磁盘格式化、裸设备写入和 fork bomb。

这些规则只提供 s01 的最低安全底线，不是完整 Shell 沙箱。更细的参数校验和人工
审批属于 s03。

### 3. 并发和取消

普通消息在 `App` 中异步启动，避免 CLI 输入循环被模型调用阻塞。`RunRegistry`
保证相同 Chat ID 同时只有一个运行：

- `/cancel` 取消当前 Agent Context。
- 模型流和 bash 命令继承该 Context。
- `/new` 先取消旧运行，再增加会话代次。
- 旧运行延迟结束时不会删除同一 Chat ID 的新运行记录。

### 4. 配置

新增配置：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `GOCLAW_MAX_STEPS` | `8` | 单次 Agent 最大模型调用次数 |
| `GOCLAW_BASH_TIMEOUT` | `10s` | 单次 bash 命令超时 |
| `GOCLAW_BASH_OUTPUT_LIMIT` | `65536` | bash 合并输出最大字节数 |

s01 启动时自动读取当前目录的 `.env`，系统环境变量优先于文件中的同名配置。启动
时会创建真实模型，因此 `LLM_API_KEY` 和 `LLM_MODEL` 必须可用。
